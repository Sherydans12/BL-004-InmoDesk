# 02 — MVP v0 Demo Comercial

## Objetivo del MVP v0

Crear una demo comercial funcional y visualmente potente para vender y validar BL-004 antes de invertir en una versión productiva completa.

No se busca todavía construir todo el sistema final. Se busca construir una demo lo suficientemente realista para que un corredor o inmobiliaria entienda el valor en una reunión.

## Flujo comercial que debe mostrar

1. El usuario entra al panel de administración.
2. Crea o edita una propiedad.
3. Completa información comercial, características, ubicación y fotos.
4. Marca la propiedad como publicada.
5. La propiedad aparece en el sitio público demo Altavista Propiedades.
6. Un interesado completa el formulario de contacto.
7. El lead aparece en el panel.
8. El agente gestiona el lead.
9. Se agenda una visita.
10. El lead avanza en el pipeline comercial.

## Pantallas mínimas del admin

- Login.
- Dashboard.
- Propiedades.
- Crear/editar propiedad.
- Detalle de propiedad.
- Leads.
- Pipeline.
- Agenda.
- Propietarios.
- Configuración pública.

## Pantallas mínimas del sitio público demo

- Home.
- Listado de propiedades.
- Detalle de propiedad.
- Publica con nosotros.
- Contacto.

## Datos mock recomendados

Para la demo conviene usar datos realistas:

- 20 a 30 propiedades.
- 5 a 8 propiedades destacadas.
- 10 a 15 leads.
- 5 visitas agendadas.
- 6 propietarios.
- 3 agentes.

### Leads

- Visitó.

- Cerrado.
- Perdido.

### Visitas

- Programada.
- Realizada.
- Cancelada.
- Reagendada.

## Criterios de aceptación de la demo

- Se puede navegar el admin sin pantallas vacías importantes.
- El dashboard muestra métricas inmobiliarias creíbles.
- El listado de propiedades tiene filtros y estados claros.
- La ficha de propiedad se ve profesional.
- La web pública demo se ve como una inmobiliaria real.
- La conexión admin → publicación pública se entiende claramente.
- El lead captado desde la web aparece en el admin o se simula de forma convincente.
- El pipeline permite mostrar seguimiento comercial.
- La agenda permite mostrar orden operativo.

## Lo que no debe pasar

- No debe parecer solo una landing.
- No debe parecer un ERP pesado.
- No debe depender del sitio público demo para explicar todo el producto.
- No debe inflarse con módulos complejos antes de validar.
