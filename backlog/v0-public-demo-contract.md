# Backlog v0 — Contrato funcional con sitio público demo

Este backlog define la relación conceptual entre BL-004 y Altavista Propiedades.

## Historias

### WEB-01 Listar propiedades publicadas

Como visitante quiero ver propiedades publicadas para explorar opciones de venta/arriendo.

Criterios:

- Solo aparecen propiedades con isPublished=true.
- Las propiedades ocultas no aparecen.
- Se muestra foto principal, precio, comuna, dormitorios, baños y superficie.

### WEB-02 Filtrar propiedades

Como visitante quiero filtrar por operación, tipo, comuna y precio.

### WEB-03 Ver detalle de propiedad

Como visitante quiero ver una ficha completa de propiedad.

Criterios:

- Se muestra galería.
- Se muestra descripción.
- Se muestran características.
- Se muestra ubicación pública, no necesariamente dirección exacta.

### WEB-04 Enviar consulta

Como visitante quiero enviar mis datos para que me contacten.

Criterios:

- Formulario incluye nombre, teléfono, email y mensaje.
- Lead queda asociado a la propiedad.
- En demo puede simularse o guardarse localmente.

### WEB-05 Captar propietario

Como propietario quiero contactar para vender o arrendar mi propiedad.

Criterios:

- Existe sección Publica con nosotros.
- El formulario genera oportunidad o lead general.
