# 01 — Alcance del producto

## Definición corta

BL-004 es un SaaS inmobiliario para administrar propiedades, interesados, propietarios, visitas y publicación web desde un panel profesional, con integración flexible hacia sitios públicos personalizados.

## Incluido en el producto base

### Panel administrativo

- Dashboard inmobiliario.
- Gestión de propiedades.
- Gestión de imágenes/galería.
- Gestión de leads/interesados.
- Pipeline comercial.
- Agenda de visitas.
- Gestión de propietarios.
- Configuración pública de empresa/corredora.
- Vista previa de publicación.

### Capa pública/integración

- API pública para propiedades visibles.
- Endpoint o mecanismo para recepción de leads.
- Código de integración para sitios externos.
- Compatibilidad conceptual con WordPress mediante bloque HTML, shortcode futuro o plugin futuro.

### Demo pública

- Sitio demo Altavista Propiedades.
- Home.
- Listado de propiedades.
- Detalle de propiedad.
- Formulario de interés.
- Sección para captar propietarios.
- Contacto.

## Excluido inicialmente

No incluir en MVP v0:

- Plugin WordPress real.
- Firma electrónica.
- Pagos online.
- Facturación.
- Publicación automática en portales externos.
- Multipublicador real.
- Tasación automática.
- IA generativa.
- App móvil nativa.
- CRM avanzado enterprise.
- Gestión legal/documental profunda.
- Contratos automáticos.
- Reportes financieros avanzados.

## Público objetivo inicial

### Prioridad 1

Corredores independientes y equipos pequeños.

Necesitan:

- Ordenar propiedades.
- Verse profesionales.
- Captar interesados desde su sitio.
- No perder leads.
- Gestionar visitas.
- Mostrar cartera actualizada.

### Prioridad 2

Corredoras pequeñas/medianas.

Necesitan:

- Varios agentes.
- Asignación de leads.
- Control de cartera.
- Métricas básicas.
- Roles.

### Prioridad 3

Inmobiliarias/desarrolladoras.

Quedan para versiones posteriores porque podrían requerir proyectos, unidades, torres, cotizaciones, financiamiento e integraciones más complejas.

## Criterio de éxito inicial

El MVP v0 será exitoso si permite mostrar en reunión un flujo completo y comprensible:

**propiedad → publicación → lead → seguimiento → visita → pipeline**.
