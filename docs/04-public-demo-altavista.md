# 04 — Sitio público demo Altavista Propiedades

## Importante

El sitio público demo **Altavista Propiedades** será un repositorio y carpeta aparte.

Este documento solo define su alcance funcional y su relación con BL-004 InmoDesk.

## Nombre de la demo

**Altavista Propiedades**

Objetivo: representar una corredora/inmobiliaria profesional ficticia conectada al panel BL-004.

## Propósito

Altavista no será el producto SaaS reutilizable. Será una vitrina comercial para mostrar cómo se puede ver un sitio público a medida conectado al admin.

Debe servir para reuniones comerciales y demostraciones en vivo.

## Estructura recomendada

### Home

Secciones:

1. Hero principal.
2. Buscador de propiedades.
3. Propiedades destacadas.
4. Venta y arriendo.
5. Por qué trabajar con nosotros.
6. Servicios para propietarios.
7. Métricas/testimonios demo.
8. CTA de contacto.

Texto conceptual:

> Encuentra tu próxima propiedad con una gestión inmobiliaria clara, profesional y cercana.

CTAs:

- Ver propiedades.
- Quiero vender o arrendar mi propiedad.

### Listado de propiedades

Debe incluir:

- Grid de propiedades.
- Filtros.
- Ordenamiento.
- Foto principal.
- Precio.
- Comuna.
- Dormitorios.
- Baños.
- Superficie.
- Estado.
- Botón ver detalle.

Filtros:

- Venta/arriendo.
- Tipo.
- Comuna.
- Precio.
- Dormitorios.
- Baños.

### Detalle de propiedad

Debe incluir:

- Galería.
- Precio.
- Ubicación aproximada.
- Características.
- Descripción.
- Mapa o bloque de ubicación.
- Formulario de contacto.
- Botón WhatsApp.
- Propiedades similares.

Formulario:

- Nombre.
- Teléfono.
- Email.
- Mensaje.

El envío debe crear o simular un lead en BL-004.

### Publica con nosotros

Sección orientada a propietarios.

Contenido recomendado:

- Evaluamos tu propiedad.
- Preparamos publicación profesional.
- Gestionamos interesados.
- Coordinamos visitas.
- Acompañamos el proceso comercial.

CTAs:

- Quiero vender mi propiedad.
- Quiero arrendar mi propiedad.

### Contacto

- Formulario general.
- WhatsApp.
- Email.
- Dirección.
- Redes sociales.

## Relación con BL-004

La demo debe poder explicar visualmente esta relación:

- El admin gestiona propiedades.
- El sitio público consume propiedades publicadas.
- El formulario público genera leads.
- Los leads vuelven al panel.

## Reglas del sitio demo

- No debe mezclar documentación interna del SaaS.
- No debe presentarse como producto principal.
- No debe condicionar el diseño de sitios futuros.
- Debe verse como una web hecha a medida para una corredora.
- Debe ser lo suficientemente profesional para vender el servicio de web pública adicional.
