# 12 — Fuera de alcance inicial

Este documento evita que el MVP se infle antes de validar comercialmente.

## No incluir en MVP v0

- Plugin WordPress real.
- Firma electrónica.
- Pagos online.
- Facturación.
- Integración automática con portales inmobiliarios.
- Multipublicación real.
- Tasación automática.
- IA.
- Chatbot.
- App móvil nativa.
- Marketplace entre corredores.
- Gestión completa de proyectos inmobiliarios.
- Contratos automáticos.
- Gestión legal avanzada.
- Reportes financieros avanzados.
- Sistema de suscripciones y billing completo.

## Permitido como simulación visual en v0

- Código de integración falso o conceptual.
- Leads generados en memoria/local/mock.
- Datos mock de propiedades.
- Dashboard con métricas calculadas desde datos demo.
- Vista pública conectada de forma simple.

## Reglas para agentes

Si un agente propone agregar funcionalidades fuera de alcance, debe dejarlo como sugerencia futura, no implementarlo sin aprobación.

Prioridad absoluta del MVP v0:

**mostrar el flujo comercial completo de manera clara y vendible.**
