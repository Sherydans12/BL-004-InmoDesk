# 11 — Principios UX/UI

## Concepto visual

El panel debe verse como una oficina inmobiliaria digital premium.

Debe transmitir:

- Confianza.
- Orden.
- Modernidad.
- Facilidad.
- Valor comercial.

## Lo que debe evitarse

- Apariencia de ERP pesado.
- Pantallas excesivamente técnicas.
- Formularios eternos sin agrupación.
- Colores demasiado oscuros como tema principal.
- Tablas saturadas.
- Lenguaje de sistema genérico.

## Estilo recomendado

- Fondo claro.
- Sidebar ordenado.
- Cards limpias.
- Imágenes grandes en propiedades.
- Estados visuales con chips.
- Formularios por secciones.
- Botones de acción claros.
- Vista previa pública.

## Lenguaje de interfaz

Usar lenguaje del negocio inmobiliario:

- Propiedad.
- Lead/interesado.
- Visita.
- Propietario.
- Publicar.
- Reservada.
- Vendida.
- Arrendada.
- En negociación.

Evitar lenguaje demasiado técnico:

- Entity.
- Record.
- Tenant en UI.
- API en pantallas comerciales salvo configuración.

## Acciones principales

Cada pantalla debe tener una acción clara:

- Dashboard: Nueva propiedad.
- Propiedades: Crear propiedad.
- Detalle propiedad: Ver publicación / Agendar visita.
- Leads: Contactar / Cambiar estado.
- Pipeline: Mover oportunidad.
- Agenda: Agendar visita.
- Configuración pública: Copiar código de integración.

## Estados visuales sugeridos

Propiedades:

- Borrador.
- Disponible.
- Reservada.
- Vendida.
- Arrendada.
- Pausada.
- Archivada.

Leads:

- Nuevo.
- Contactado.
- Visita agendada.
- Visitó.
- En negociación.
- Cerrado.
- Perdido.
